# API Documentation

## Base URL
- Local: `http://127.0.0.1:8000`
- Hosted: replace with your deployed URL

## Authentication
### POST `/auth/register`
Creates a new user account.

Example request:
```json
{
  "name": "Alice",
  "email": "alice@example.com",
  "password": "secret123"
}
```

### POST `/auth/login`
Returns a bearer token.

Form data:
- `username`: user email
- `password`: plain-text password

Example response:
```json
{
  "access_token": "<jwt>",
  "token_type": "bearer"
}
```

## Books
### POST `/books`
Creates a book record.

### GET `/books`
Returns all books. Optional query parameters:
- `genre`
- `author`

### GET `/books/{book_id}`
Returns a single book.

### PUT `/books/{book_id}`
Updates a book.

### DELETE `/books/{book_id}`
Deletes a book.

## Users
### GET `/users`
Lists users.

### GET `/users/{user_id}`
Returns a user.

## Reviews
All write operations require `Authorization: Bearer <token>`.

### POST `/reviews`
Creates a review owned by the authenticated user.

Example request:
```json
{
  "book_id": 1,
  "rating": 5,
  "comment": "Excellent architecture book"
}
```

### GET `/reviews`
Lists all reviews.

### PUT `/reviews/{review_id}`
Updates a review. Only the owner can modify it.

### DELETE `/reviews/{review_id}`
Deletes a review. Only the owner can delete it.

## Analytics
### GET `/analytics/top-rated-books`
Returns highest-rated books.

### GET `/analytics/genre-distribution`
Returns count of books by genre.

### GET `/analytics/most-reviewed-books`
Returns books with the most reviews.

### GET `/analytics/books-per-year`
Returns publication-year counts.

### GET `/analytics/recommendations/{user_id}`
Rule-based recommendations based on the user's strongest-rated genre.

## Status codes used
- `200 OK`
- `201 Created`
- `204 No Content`
- `400 Bad Request`
- `401 Unauthorized`
- `403 Forbidden`
- `404 Not Found`
- `422 Unprocessable Entity`
