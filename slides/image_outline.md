This deck presents a public-data-driven book metadata and recommendation API that evolved into a provenance-aware discovery platform. The audience is coursework assessors, so the visuals should feel polished, credible, and technically ambitious without looking like default corporate slides.

Slide 1 - Title and framing
Create an elegant opening visual for a modern book-discovery platform. Combine a curated bookshelf atmosphere, subtle interface fragments, metadata motifs, and calm empty space for a title block. The mood should be academic, confident, and editorial rather than playful.

Slide 2 - Problem, motivation, and originality
Create a conceptual visual showing reader trust, content overload, and content-origin uncertainty in a book or web-novel ecosystem. Suggest contrast between clean curated reading and noisy low-trust content, with generous negative space for explanatory cards.

Slide 3 - Architecture and technology stack
Create a modular systems visual for an API platform with abstract service blocks, database layers, auth flow, analytics flow, and migration pathways. The composition should feel architectural and structured, leaving room for an editable stack diagram.

Slide 4 - Public dataset integration and provenance model
Create a data-centric visual blending books, metadata streams, tags, provenance labels, moderation markers, and translation cues. It should feel like public catalogue data being enriched by a governance layer, without any readable labels.

Slide 5 - Recommendation and similar-book discovery
Create a visual for explainable recommendation and provenance-aware similar-book discovery. Use branching paths, highlighted content cards, rating cues, and trust-filter motifs. Keep broad calm areas for an editable workflow explanation.

Slide 6 - Engineering quality
Create a reliability-focused visual that suggests testing, validation, migration checkpoints, and API quality. Use abstract charts, check markers, pipeline geometry, and technical polish, while staying text-free and presentation-ready.

Slide 7 - Outcome and demo flow
Create a concluding visual that feels like a live demo handoff: a refined analytics/dashboard atmosphere combined with books, discovery cues, and a sense of project completion. Leave a strong calm region for the final summary and demo steps.
