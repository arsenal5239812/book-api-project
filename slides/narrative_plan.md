# Book Metadata and Recommendation API Presentation Plan

## Audience
Module tutors evaluating the coursework submission during presentation and oral Q&A.

## Objective
Show that the project is not a basic CRUD API, but a public-data-driven, provenance-aware discovery platform with strong engineering discipline.

## Narrative arc
1. Establish the coursework problem and why a book API is a good fit.
2. Show the system architecture and technology choices.
3. Demonstrate how public data from `goodbooks-10k` is integrated into the platform.
4. Introduce the provenance and AI-transparency layer as the distinctive project theme.
5. Explain how recommendation and similar-book discovery use reader preference and provenance metadata.
6. Prove engineering maturity through testing, migrations, and documentation.
7. Close with the final outcome and demo flow.

## Slide list
1. Title and project framing
2. Problem, motivation, and originality
3. Architecture and technology stack
4. Public dataset integration and provenance model
5. Recommendation and similar-book discovery
6. Engineering quality: testing, validation, and migrations
7. Submission outcome and live demo pathway

## Source plan
- `report/technical_report.md`
- `docs/api_documentation.md`
- `README.md`
- Local codebase structure and implemented endpoints

## Visual system
- Clean editorial look with warm ivory background, deep charcoal text, emerald accent, and muted gold highlights.
- One text-free art-direction plate per slide with calm space for editable content.
- Use structured cards, simple diagrams, and one native chart-style slide for engineering evidence.
- Typography:
  - Titles: `Caladea`
  - Body: `Lato`
  - Technical labels: `Aptos Mono`
